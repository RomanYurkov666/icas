POST /adÂ¶

This URL creates a new ad for the current user. If the ad was successfully created then a 201 Created is returned and the Location header contains the URL to the newly created ad. The optional query parameter _body with value true or false can be used to influence whether the response body contains the ad in the body (or not) in case of success. By default, this value is true. If the ad contains errors then the server returns 400 Bad Request with a list of errors.

For a new ad the id attribute must be omitted or set to zero. The title and description must be set and will be trimmed. The salutation contains either MALE, FEMALE, COMPANY or FAMILY. The sellerName contains the name of the seller as displayed on the site.

The categoryId must be an existing category id from a category which has canPlaceAds set to true. If the status of the ad is ACTIVE and the current number of active ads for the same user in this category exceeds maxNumberOfActiveAds as defined in the category then an error is returned. Ads with status equals to PAUSED do not have this restriction.

The currency must be set to EUR and all price amounts must be specified in euro cents.

The priceType must be one of the valid types listed in Price Types. If the priceType is either FIXED_PRICE or BIDDING_FROM then a price greater than zero must be specified. For all other price types price must be omitted.

A new ad must have a status which can be either ACTIVE or PAUSED. An active ad is shown on the site whereas a paused ad is not. See Ad Status for the list of valid ad status values.

The cpc is a mandatory field whose value depends on the category. The totalbudget and dailyBudget are optional fields whose value depend on the category. Omitting them will remove budget limits for this ad. The category configuration which can be retrieved through the category URL contains the valid bounds for these fields once filled in.

All prices are specified as numbers, i.e. a price of 10,50 EUR must be specified as 1050. This applies to price, totalBudget and dailyBudget.

The postCode is optional. In case it is not specified, its value is set to Heel Nederland.

The images field is optional and can contain a list of up to 8 Image Objects which contain the URL of the image on the callers server.

The attributes field is optional and can contain a list of User-Defined Attributes.

In order to use ‘Buy It Now’ Sellside API users need to have the ‘Buy It Now’ permission. When buyItNowEnabled field is set to true, paypalEmail field must contain a valid email address, priceType must be FIXED_PRICE and price must be greater than 1. If buyItNowEnabled field is not set or set to false paypalEmail is ignored.

ParametersÂ¶

Name Type Default Description
_body boolean true Include the created ad in the response body on success. Disabling this will speed up the call.
_include string “” Comma-separated-list of fields to include. Optional, default is all fields.
_exclude string “” Comma-separated-list of fields to omit. Optional, default empty.
